## 二、技术架构设计

### 1. 前端技术栈

#### 核心框架
- **Vue 3**: 采用Composition API，支持TypeScript
- **Vite**: 快速构建工具
- **TypeScript**: 类型安全
- **Pinia**: 状态管理
- **Vue Router**: 路由管理

#### UI组件库
- **Arco Design Vue**: 主要UI组件库（已使用）
- **Element Plus**: 备选方案（需求文档提及，但项目已使用Arco Design）

#### 数据可视化
- **ECharts**: 图表渲染（已集成）
- **自定义组件**: 雷达图、饼图、折线图

#### 其他工具
- **Axios**: HTTP请求
- **VueUse**: 组合式API工具集
- **Markdown-it**: Markdown渲染

### 2. 后端技术栈

#### 核心框架
- **FastAPI**: 高性能Web框架
- **SQLAlchemy**: ORM
- **Alembic**: 数据库迁移
- **Pydantic**: 数据验证

#### AI集成
- **LangGraph**: Agent框架
- **MCP (Model Context Protocol)**: 工具调用协议
- **DeepSeek**: 大语言模型

#### 数据库
- **PostgreSQL**: 主数据库
- **Redis**: 缓存（可选）

### 3. API设计规范

#### RESTful API
```
GET    /api/v1/courses          # 获取课程列表
GET    /api/v1/courses/:id      # 获取课程详情
POST   /api/v1/chat/threads     # 创建对话
GET    /api/v1/chat/history     # 获取对话历史
```

#### SSE流式接口
```
GET    /api/v1/chat/stream      # SSE流式对话
```

#### 文件上传
```
POST   /api/v1/rag/upload       # 上传参考文件
```

## 五、技术难点与解决方案

### 1. 实时视频流延迟问题

**难点**: WebRTC需要信令服务器，HLS延迟较高

**解决方案**:
- 使用WebRTC进行低延迟直播（< 1秒）
- 备选HLS用于高兼容性场景（延迟3-5秒）
- 实现自适应码率切换

### 2. YOLO识别框实时渲染

**难点**: 需要高频率更新识别框位置

**解决方案**:
- 使用Canvas进行高效渲染
- 实现识别框平滑动画
- 优化数据传输（只传输变化区域）

### 3. SSE流式响应中断处理

**难点**: 网络不稳定导致连接中断

**解决方案**:
- 实现自动重连机制
- 添加心跳检测
- 断点续传功能

### 4. 大文件上传处理

**难点**: 参考文件可能很大

**解决方案**:
- 实现分片上传
- 添加上传进度显示
- 支持断点续传

### 5. AI上下文长度限制

**难点**: 视频内容+对话历史可能超出Token限制

**解决方案**:
- 实现智能摘要
- 分段处理长内容
- 使用向量检索相关内容
