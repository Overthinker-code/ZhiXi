# 行为检测模块修改上传说明

> 生成时间：2026-04-16  
> 涉及范围：YOLO 服务、后端 API、前端实时监控页面

---

## 一、变更目标

1. **摄像头 URL 统一后端管理**：前端不再硬编码教室摄像头地址，改为从后端 `/api/behavior/cameras` 动态读取。唯一手动配置点：`behavior_analysis.py` 的 `get_classroom_cameras()` 函数。
2. **消除行为定义多源头不一致**：后端 `/behaviors/definitions` 改为直接转发 YOLO 服务定义，避免 3 套不同定义互相冲突。
3. **修复前端运行时 Bug**：`behavior-detection-panel.vue` 中移除了未定义函数 `drawDetectionBoxesOnVideo()` 的调用。
4. **升级默认检测模型**：YOLO 服务默认检测器从 `yolov8n.pt` 升级为 `yolo11m.pt`（教室场景精度更优）。

---

## 二、详细变更清单

### 1. `code/cv/code/yolo.py`

**修改点**：
- `DETECTOR_PATH` 默认值从 `../model/yolov8n.pt` 改为 `../model/yolo11m.pt`。
- `DETECTOR_CONF` 默认值从 `0.3` 改为 `0.35`。

**影响**：
- 启动 YOLO 服务时如不指定环境变量，自动使用 `yolo11m.pt`。
- 若 `yolo11m.pt` 不存在，服务会**自动回退到单阶段模式**（仅使用 `yolov8n-pose.pt`），不影响可用性。

**重启要求**：✅ 必须重启 YOLO 服务（端口 8001）

---

### 2. `code/backend/app/api/routes/behavior_analysis.py`

**修改点**：
1. `/cameras` 接口：添加醒目的手动配置注释，明确告知维护者此处是唯一需要修改摄像头地址的地方。
2. `/behaviors/definitions` 接口：删除原来的硬编码行为定义，改为 `await behavior_service.get_behavior_definitions()`，即从 YOLO 服务 `/behaviors` 接口原样转发。

**影响**：
- 前后端行为定义、颜色、评分区间完全统一。
- 新增行为或修改评分规则时，只需改 `yolo.py` 一处。

**重启要求**：✅ 必须重启后端服务（端口 8000）

---

### 3. `code/backend/app/services/behavior_analysis.py`

**修改点**：
- 删除死代码 `_evaluate_learning_status(self, score)` 方法（从未被调用）。

**影响**：
- 消除与 YOLO 服务内评分逻辑不一致的误导性代码。

**重启要求**：✅ 随后端服务重启生效

---

### 4. `code/education/course/src/api/behavior-analysis.ts`

**修改点**：
- 新增 `getClassroomCameras()` 函数，调用后端 `/api/behavior/cameras`。

**影响**：
- 前端获取远程摄像头地址的标准接口。

**重启要求**：❌ 前端 Vite HMR 自动生效，无需重启

---

### 5. `code/education/course/src/views/course/monitor/index.vue`

**修改点**：
1. **移除硬编码 `cameraUrl`**：`monitorCourses` 数组中不再包含摄像头地址。
2. **动态拉取配置**：`onMounted` 时调用 `getClassroomCameras()`，将地址映射到 `cameraUrlMap`。
3. **选择远程摄像头时校验**：`onSelectClassroomCamera()` 先从 `cameraUrlMap` 取地址，取不到则提示用户检查后端配置。
4. **统一颜色映射**：`getScoreColor()` 优先读取后端 `/behaviors/definitions` 返回的 `score_ranges`，失败时走默认兜底颜色。
5. **加载行为定义**：`onMounted` 时同步调用 `getBehaviorDefinitions()` 缓存到 `behaviorDefinitions`。

**影响**：
- 前端页面打开后自动获取最新摄像头地址和行为定义；改摄像头地址无需重新构建前端。

**重启要求**：❌ 前端 Vite HMR 自动生效，浏览器刷新即可

---

### 6. `code/education/course/src/views/course/monitor/components/behavior-detection-panel.vue`

**修改点**：
1. **修复运行时 Bug**：`onResize` 中删除了不存在的 `drawDetectionBoxesOnVideo()` 调用，避免窗口 resize 时控制台报错。
2. **统一颜色映射**：`getScoreColor()` 同步改为优先读取后端 `score_ranges`。

**影响**：
- 上传图片/视频分析时的侧边面板行为检测展示更稳定，颜色逻辑与主舞台保持一致。

**重启要求**：❌ 前端 Vite HMR 自动生效，浏览器刷新即可

---

### 7. `code/cv/` 目录结构变更

**变更点**：
- YOLO 相关代码整体迁移到 `code/cv/code/`。
- 模型文件存放于 `code/cv/model/`（`yolov8n-pose.pt`、`yolov8n.pt`、`yolo11m.pt`、`yolo11l.pt`）。
- 原 `code/yolo.py`、`code/yolo_integrated.py` 已废弃。

**影响**：
- 项目结构更清晰，CV 模块独立维护。

## 三、手动配置点说明

### 唯一需要手动修改摄像头地址的位置

文件：`code/backend/app/api/routes/behavior_analysis.py`

函数：`get_classroom_cameras()`

示例：
```python
return {
    "cameras": [
        {"id": "db",  "cameraUrl": "http://192.168.1.101:8080/video"},
        {"id": "ds",  "cameraUrl": "http://192.168.1.102:8080/video"},
        {"id": "ai",  "cameraUrl": "http://192.168.1.103:8080/video"},
        {"id": "eco", "cameraUrl": "http://192.168.1.104:8080/video"},
    ]
}
```

**格式要求**：浏览器 `<video>` 标签不支持裸 RTSP，请使用 HTTP / MJPEG / FLV / HLS(m3u8) 格式。

## 四、部署/重启步骤

按顺序执行以下操作：

1. **确认模型存在**
   ```powershell
   dir D:\ZhiXi\code\cv\model\*.pt
   # 至少应有 yolov8n-pose.pt 和 yolo11m.pt
   ```

2. **重启 YOLO 服务**
   ```powershell
   cd D:\ZhiXi\code\cv\code
   python -m uvicorn yolo:app --host 0.0.0.0 --port 8001
   ```
   确认日志输出：
   - `✅ 人体检测模型加载成功：../model/yolo11m.pt`
   - `🚀 已启用 YOLO11 检测器，教室场景检测能力更强`

3. **重启后端服务**
   ```powershell
   cd D:\ZhiXi\code\backend
   python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```

4. **刷新前端页面**
   - 浏览器打开课堂监控页，按 `F5` 刷新即可。
   - 首次打开时会自动请求 `/api/behavior/cameras` 和 `/api/behavior/behaviors/definitions`。
