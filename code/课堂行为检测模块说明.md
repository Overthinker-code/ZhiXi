# 功能模块技术与逻辑说明

## 模块名称
课堂行为检测模块（YOLOv8-Pose 行为识别）

## 负责人
wjzhang

## 实现功能
基于 YOLOv8-Pose 姿态估计和几何规则，实时识别学生课堂行为（专注学习/查看手机/与他人交谈/睡觉/离开座位）。

---

## 核心逻辑

### 1. 输入
- **图像输入**：视频帧或图片（来自摄像头或用户上传）
- **关键点序列**：连续 16 帧的人体关键点（17 个关键点 × 2 维坐标）
- **单帧关键点**：单张图片中检测到的人体姿态（17 个关键点）

### 2. 处理流程
```
输入图像/视频帧
    ↓
阶段1: YOLOv8-Pose 姿态估计
  - 检测图片中所有人
  - 提取 17 个关键点（鼻子、眼睛、肩膀、手腕等）
  - 输出关键点坐标 (x, y)
    ↓
阶段2: 关键点预处理
  - 归一化坐标到 [0,1] 范围
  - 筛选有效检测（置信度 > 0.5）
    ↓
阶段3: 规则-Based 行为分类
  ├─ 看手机：手腕靠近面部（距离<0.15）且头部低下
  ├─ 睡觉：头部低于肩部（低头）或后仰
  ├─ 交谈：头部转向侧面（耳高差异>0.06）
  ├─ 离开：人体检测不完整（关键点<8个）
  └─ 专注：默认正常姿态
    ↓
阶段4: 时序平滑（视频/实时流）
  - 滑动窗口分析（16帧窗口，50%重叠）
  - 统计各行为出现频次
  - 输出最可能的行为类别
    ↓
阶段5: 统计分析
  - 计算综合评分（各行为加权平均）
  - 统计各类行为人数
  - 评估整体学习状态
```

### 3. 输出
- **行为识别结果**：
  - 行为类别（专注/手机/交谈/睡觉/离开）
  - 置信度（0-1）
  - 判断依据（如"手腕靠近面部"）
- **学习状态评分**：
  - 综合得分（-1.0 ~ +1.0）
  - 状态等级（优秀/良好/一般/较差/极差）
- **统计信息**：
  - 检测人数
  - 各类行为人数分布
  - 教室整体学习状态

## 关键实现点

### 1. YOLOv8-Pose 姿态估计
使用预训练的 YOLOv8n-Pose 模型检测人体 17 个关键点（COCO 格式），包括鼻子、眼睛、耳朵、肩膀、手肘、手腕、髋部、膝盖、脚踝等，输出每个关键点的 (x, y) 坐标。

### 2. 规则-Based 行为分类器
基于人体关键点的几何关系设计判断规则：
```python
# 看手机判断逻辑
wrist_to_nose_dist < 0.15 and head_low  # 手腕靠近面部 + 低头

# 睡觉判断逻辑  
nose_y > shoulder_center_y + 0.12  # 头部低于肩部

# 交谈判断逻辑
abs(left_ear_y - right_ear_y) > 0.06  # 头部转向，耳高差异大
```

### 3. 滑动窗口时序平滑
对视频流使用滑动窗口（16帧，50%重叠）进行时序分析，统计窗口内各行为出现频次，取最频繁的行为作为该时段识别结果，减少单帧抖动。

### 调用关系说明

| 调用方 | 被调用方 | 调用方式 | 说明 |
|--------|----------|----------|------|
| 前端面板 | 后端 API | HTTP POST/GET | 上传图片/获取结果 |
| 后端 API 路由 | 业务服务层 | 内部函数调用 | 封装 YOLO 调用 |
| 业务服务层 | YOLO 服务 | HTTP API | 发送图片到 YOLO 服务 |
| YOLO 服务 | YOLOv8-Pose 模型 | 本地调用 | 姿态估计推理 |
| YOLO 服务 | 规则分类器 | 本地调用 | 行为识别计算 |
