**目标**：1天内完成数据准备和模型训练，准确率 **75-85%**

## 🎬 第一步：录制视频数据（2-3小时）

### 目录结构

创建 `data/` 目录，按以下结构放入视频：

```
data/
├── 0_focused/          # 专注学习（正常坐姿看书/屏幕）
├── 1_phone/            # 看手机（手举着手机，低头）
├── 2_talking/          # 交谈（转头说话）
├── 3_sleeping/         # 睡觉（趴在桌上或仰头闭眼）
└── 4_away/             # 离开（起身离开画面）
```

### 录制要求

| 类别 | 录制要点 | 建议数量 | 时长 |
|------|---------|---------|------|
| **专注学习** | 正常坐着看书或屏幕，头正、手在桌上 | 10-20个 | 3-5秒 |
| **看手机** | 手举着手机，头低下，眼睛看手机 | 10-20个 | 3-5秒 |
| **交谈** | 头转向侧面，嘴巴动（模拟说话） | 10-20个 | 3-5秒 |
| **睡觉** | 趴在桌上，或仰头闭眼 | 10-20个 | 3-5秒 |
| **离开** | 起身，走出画面 | 10-20个 | 3-5秒 |

### 录制技巧

1. **光线充足**：确保面部和上半身清晰可见
2. **背景简单**：避免杂乱背景干扰
3. **多角度**：每个类别换不同角度录几个
4. **不同衣服**：换几件衣服录，增加多样性
5. **手机录制即可**：1080p，30fps，不需要专业设备

### 命名示例

```
data/
├── 0_focused/
│   ├── video_01.mp4
│   ├── video_02.mp4
│   └── ... (10-20个)
├── 1_phone/
│   ├── phone_01.mp4
│   ├── phone_02.mp4
│   └── ... (10-20个)
└── ...其他类别
```

## 🚀 第二步：一键训练（自动完成）

### 1. 确保依赖已安装

```bash
pip install ultralytics opencv-python torch torchvision numpy scikit-learn tqdm
```

### 2. 运行训练脚本

```bash
python quick_collect_and_train.py --data_dir ./data --epochs 50
```

**脚本会自动完成：**
1. ✅ 提取所有视频的姿态关键点
2. ✅ 自动数据增强（时序抖动、空间噪声、翻转等）
3. ✅ 划分训练集和验证集
4. ✅ 训练LSTM模型
5. ✅ 自动早停（防止过拟合）
6. ✅ 保存最佳模型
7. ✅ 输出分类报告

## 🔧 参数调优（可选）

### 如果准确率不理想

**情况1：训练准确率很高，验证准确率低（过拟合）**
```bash
# 减少训练轮数
python quick_collect_and_train.py --epochs 30

# 或增加数据量（每类至少15个视频）
```

**情况2：整体准确率都低（欠拟合）**
```bash
# 增加训练轮数
python quick_collect_and_train.py --epochs 100

# 或增加隐藏层大小（需要改代码）
```

**情况3：某类识别很差**
- 检查该类视频质量
- 增加该类视频数量
- 确保标签正确

### 高级参数

```bash
python quick_collect_and_train.py \
    --data_dir ./data \
    --output_dir ./my_model \
    --epochs 50 \
    --batch_size 32 \
    --seq_length 16 \
    --device cuda  # 或 cpu
```

## ⚠️ 常见问题

### Q1: 需要多少视频才能训练？
**A**: 最少每类5个（共25个），推荐每类10-20个（共50-100个）。

### Q2: 视频必须3-5秒吗？
**A**: 不必须，但太短（<2秒）可能提取不到足够帧，太长（>10秒）处理慢。

### Q3: 可以多人一起录吗？
**A**: 可以！多人录制可以增加数据多样性，提高模型泛化能力。

### Q4: 必须是mp4格式吗？
**A**: mp4、avi、mov都可以。

### Q5: 训练时显示CUDA out of memory？
**A**: 减小batch_size：
```bash
python quick_collect_and_train.py --batch_size 16
```

### Q6: 没有GPU怎么办？
**A**: CPU也可以训练，只是慢一些（1-2小时）：
```bash
python quick_collect_and_train.py --device cpu
```

### Q7: 训练完成后怎么优化？
**A**: 
1. 查看混淆矩阵，找出容易混淆的类别
2. 针对这些类别增加更多训练数据
3. 重新训练
