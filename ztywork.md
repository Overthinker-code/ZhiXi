# 知曦系统 - 前端实现情况与缺陷分析大纲 (2026-04 更新版)

## 🌟 一、当前已实现的核心模块与视觉交互归纳

经过多轮密集开发，项目前端在“Apple 设计风格（毛玻璃、平滑过渡、极简配色）”的指引下，已完成大量核心业务组件与数据可视化页面的建设。当前系统已具备的页面及功能能力如下：

### 1. 登录与基础框架 (Day 1-2 遗留成果)
- **多合一认证页**：登录、注册、密码恢复的三合一表单切换，已接入基础 API 并包含校验拦截。
- **全局三态组件**：全局封装的 `<LoadingState />`、`<EmptyState />`、`<ErrorState />` 加载组件，各页面已统一使用。
- **路由/状态管理**：基于 Pinia 的权限守卫，路由动态加载。

### 2. 课堂监控大屏 (P0 核心升级)
- **出勤监控 (`attendance-grid.vue`)**：实现了 18 宫格动态学生名牌，支持按“已签到/迟到/缺席/请假”状态筛选、5s 自动轮询、带有毛玻璃效果的 Hover 学生详情悬浮层，以及出入场动画。
- **视频 AI 监测 (`studio.vue`)**：重构了直播核心监控区。底层采用 `<video>` 元素，上层依托 ResizeObserver 在 `<canvas>` 上精确叠加渲染 YOLO 识别框坐标，200ms 高频更新并带有平滑过滤；集成了流参数（码率/帧率）及画质/行为检测开关功能。

### 3. 课程资源与互动 (P0 核心升级)
- **知识点交互视频 (`coursevideo/index.vue`)**：创新实现了将视频进度与知识点“时间轴”强绑定。拥有平滑折线渐变的 ECharts 知识点重要度走势图；右侧 AI 伴学抽屉能够在当前视频时间戳节点基于上下文进行直接发问。

### 4. 智能助手体系 (P1 核心能力)
- **多维参数控制 (`SettingsPanel.vue`)**：重构为高级助手控制台。支持大型模型快捷切换、流式开关、精准控制的大语言生成滑块（Temperature / Max Tokens等），并加入了对 MCP（模型上下文协议）工具集合的多选与 beta 级状态追踪。
- **Tool Calling 可视化**：打通了 `useChat` 到底层 SSE 事件读取的信道，能够在助手响应时悬浮或嵌套展现工具调用进度（闪烁动画与阶段提示）。

### 5. 学情分析看板 (P1 核心能力)
- **多维分析视图 (`user/study/index.vue`)**：摒弃传统硬编码，使用 Composition API 和 Arco Design 重构。包含蓝绿色渐变的学生画像卡片、6 格高斯指标网格。
- **可视化与预警**：中心带有数据总汇的 ECharts 时长分布环形图，并配有区分警告等级（橙色/红色边界）的预警列表与日历即将到期事件时间轴。

### 6. 数据中心与附加组件 (P2 扩展功能)
- **课程作业排行榜 (`leaderboard/index.vue`)**：含前三名阶梯颁奖台（皇冠/奖牌飘动特效，金银铜定制渐变）、支持成绩/完成率双轨排序的动态 Arco 列表、详情抽屉里的 AI 学情点评面板，并支持导出 CSV。
- **思维导图渲染器 (`MindMapViewer.vue`)**：集成了 `markmap`，提供从 Markdown 文本到可交互缩放、可下载 SVG 的思维导图转换引擎。

## 🔍 二、前端当前依然存在的缺漏与未完成拼图

尽管上述页面结构和功能逻辑已经搭建完成，但距离一个“完全体”仍有最后 15% 的关键断层：

### 1. 从“假数据”到“真接口”的全面挂载（当前最大鸿沟）
目前绝大多数新页面（监控大屏、视图统计、学情看板、排行榜）都仅预留了 TypeScript Interface 定义和带有 `// TODO: GET /api/...` 注释的 Mock 数据集。
**缺漏点**：一旦后端（张伟杰）交付了真实的 API 端点，前端必须花至少 1-2 天时间进行大量的 Axios 返回值替换与 Edge case 控制（500 超时报错处理）。

### 2. 知识图谱功能（待实现）
在初始规划中提到的网格化知识点呈现、连线点击。目前只完成了基于层级 Markdown 的“思维导图”渲染（Markmap），缺少基于节点和连线（用 G6 或 D3.js 实现）的“知识图谱大纲”拖拽视图。

### 3. 新建组件的路由挂载缺失
第二、第三轮快速冲刺完成了思维导图 (`MindMapViewer.vue`) 与 排行榜页面 (`leaderboard/index.vue`)。其中排行榜已经配有路由，但思维导图作为一个基础公共组件，还没有挂载到指定的业务入口（例如在课程详情中的某个 Tab 下可以调用该渲染器）。

### 4. 操作回路中断：缺少“作业提交系统”
排行榜显示了同学们作业的情况，但该系统中缺少一个“学生真实上传文件、提交作业、填写备注”的表单页面。系统仅有看榜入口，缺少产出入口。

- 保证 `/chat/` 推流时的连接稳定性。
